# Bone 主题验收样张

这一份文档把 Typora 会渲染的元素基本铺满了，用来逐项验收排版。中英混排是重点：西文走 Anthropic Sans，中文自动 fallback 到苹方——**这不是妥协，桌面端本身就是这个机制**。

## 一、正文与中英混排

我写这段是为了看长段落的呼吸感。行高定在 1.78，段间距 1.15rem，行宽 46rem。判断标准很朴素：连续读三段不累眼，中英文交界处不出现明显的基线错位或字重突变。

比如这句里混着 `variable font`、opsz 轴、以及 300→800 的字重范围，看看 Anthropic Sans 和苹方交接得顺不顺。数字对齐也要看：2026 年、1,247 次、99.8% 的占比。

行内样式一次过：**加粗看字重是不是真的**、*斜体看有没有真斜体而不是机器倾斜*、~~删除线~~、`行内代码`、[一个链接](https://example.com)、以及 ==高亮==。

### 三级标题：看字阶是否连贯

标题的负字距是高级感的主要来源，字号越大收得越多。h1 到 h4 应该形成清晰但不突兀的层级。

#### 四级标题

到这一级应该已经接近正文字号，靠字重和间距区分，而不是继续放大。

```python
def check(css):
    """判据必须能分辨合法与违规，一刀切等于没有区分力。"""
    problems = []
    for m in re.finditer(r"([^{}]+)\{([^{}]*)\}", css):
        selector = m.group(1).strip().split("\n")[-1].strip()
        if selector == "html":
            continue                      # 合法：根字号
        for fm in re.finditer(r"font-size:\s*[\d.]+px", m.group(2)):
            problems.append(f"{selector} 里 {fm.group(0)}")
    return problems
```
