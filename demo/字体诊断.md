# 字体诊断

重启 Typora → 主题选 Bone → 打开这一篇，看下面三行。

<p style="font-family:'Bone Sans',sans-serif;font-size:30px;margin:0 0 6px">A　Handgloves Fupanwang 2026 &nbsp; agRSK</p>
<p style="font-family:-apple-system,BlinkMacSystemFont,sans-serif;font-size:30px;margin:0 0 6px">B　Handgloves Fupanwang 2026 &nbsp; agRSK</p>
<p style="font-size:30px;margin:0 0 6px">C　Handgloves Fupanwang 2026 &nbsp; agRSK</p>

**怎么判断**（A 是强制主题字体，B 是强制系统字体 SF Pro，C 是正文默认）：

| 现象 | 结论 |
|---|---|
| A 和 B 长得**一模一样** | 字体没加载上，A 悄悄 fallback 成了 SF Pro |
| A 和 B **明显不同**，且 C 跟 A 一样 | ✅ 正常，正文用上了 Anthropic Sans |
| A 和 B 不同，但 C 跟 B 一样 | 字体加载了，但正文没用上（字体栈顺序问题） |

看 `a` `g` `R` `K` 这几个字母的字形差异最明显。

## 字重梯度

如果上面 A ≠ B（说明加载成功），再看这四行是不是**逐级变粗**：

<p style="font-family:'Bone Sans';font-weight:300;font-size:26px;margin:0">300　Handgloves 2026　可变轴下限</p>
<p style="font-family:'Bone Sans';font-weight:400;font-size:26px;margin:0">400　Handgloves 2026　正文常规</p>
<p style="font-family:'Bone Sans';font-weight:650;font-size:26px;margin:0">650　Handgloves 2026　加粗档</p>
<p style="font-family:'Bone Sans';font-weight:800;font-size:26px;margin:0">800　Handgloves 2026　可变轴上限</p>

四行若粗细无差别，说明 wght 轴没生效（那就是在合成假粗，边缘会发虚）。

## 中英混排（最终要看的就是这个）

调研日期 2026-07-20，系统 UniApp，插件 SDK 三合一，README.md 里写着 `nativePlugins` 是空的。付了 1200 块买导航插件，查完才发现它大概率一天都没被用上。

这一段是**真实的混排场景**：中文走苹方、西文走 Anthropic Sans、代码走等宽。重点看中英文交界处有没有基线错位或字重突变。
